Chapter 4 - Fractions or "lazy divisions"

I will not calculate this division!

We could do like the ancient Greeks and keep divisions un-calculated ... or at least keep them not solved as long as posible in our calculations. This is known as lazy evaluation and it will lead us to fractions.
First let's see how we can avoid calculating divisions as long as possible in a mathematical expression. Then we can give a special syntax to an un-calculated division, ... and a better name: a fraction.

Procrastinate... in a while

Let's say that I want to AVOID calculating divisions as long as possible in a mathematical expression, for example in: (10:2) + (20:2) The usual way to go is to first calculate the two divisions (in the two brackets), then add; so:

(10:2) + (20:2)

5 + 10
⇓ +
15
But to delay the division as much as possible, I could try to do the addition first, then the division... if I really have to. I woudl get something like:
(10:2) + (20:2)
⇓ +
???

OK, but wait a second! How can I add to divisions, before I even calculate them?
Well, I could look at what each division means: 10:2 means "take ten sheeps and divide them equally between two of my friends", and 20:2 means "take twenty sheeps and divide them equally between the same two of my friends". The two friends will get the sum of all those sheeps, divided between them.
However you calculate this, it must be that each friend gets half of thirty sheeps, right?! But that means that: (10:2) + (20:2) = (10+20):2 = 30:2, and this is only the case because I am dividing my sheeps among the same two friends. It would not work for something like (10:2) + (20:3). But back to our alternative way to calculate the expression:
(10:2) + (20:2)
⇓ +
(10+20):2 = 30:2
⇓ :
15

Nice! It does not matter in which order I do the calculations, I stil get the same result 15. The advantage in the second case is that I have to do only one division, which is potentially a good, lazy way to go... and perhaps I could even avoid that one! Would it be wrong to say: (10:2) + (20:2) = 30:2 and stop there, without giving the final result as 15? Can 30:2 be considered kind of like a number by itself? Not really, mainly because it is clear that thirty is divisible by two, so I should do better than stop here, and finish my calculation.
Fine, sure. But there might be other cases where I could prefer to stop at the division, like for example 1:2. Perhaps here it would be better to leave 1:2 as a result of some expression, instead of incomodating 0.5, which is a broken number (see the previous chapter) and not so nice to look at anyways... And if I decide to go this way, I need to find out what happens with I add an un-calculated division like 1:2 to a whole number, or when I multiply it by some other number or even by another un-calculated division:
  • (1:2) + 3 = ?
  • (1:2) * 3 = ??
  • (1:2) * (3:4) = ???
because so far the only thing we know is that (a:b) + (c:b) is equal to (a+c):b.

The un-calculated divisions

The first puzzle was (1:2) + 3. Unfortunately I don't have a rule for this one, and so far I only know how to add a number to another number, and an un-calculated divisions to another, but only when they are similar enough (which means that both divide by the same number, like 10:2 and 20:2). What can I do here? If only I could rewrite three as a division by two, then I could easily calculate the addition... So I would like to rewrite 3 as something:2, but there is a number like that, it is 6. Remember our idea that in mathematics we keep going back to the fact that the same meaning can be expressed in many ways (semantics versus syntax) and 3 is actually also 6:2, or 9:3 by the way. Following this line of thought I can say that any whole number m is also equal to (AKA has the same meaning as) any un-calculated division that looks like (m*n):n, provided that n is not zero, which would create problems with the division.
OK, good, so the trick here could be: (1:2) + 3 → (1:2) + ((3*2):2) = (1:2) + (6:2)

and now I can use my rule for adding un-calculated divisions: (1:2) + 3 = (1:2) + (6:2) = (1+6):2 = 7:2 which eventually would be 3.5.
But is this the correct result? Let's see with a calculation diagrams:
(1:2) + 3
⇓ :
0.5 + 3
⇓ +
3.5
(1:2) + 3
⇓ +
1:2+((3*2):2) = (1+6):2 = 7:2
⇓ :
3.5
good! So what is the rule in general? (a:b) + n = (a:b) + ((n*b):b) = (a + n*b):b with a, b, and n being whole numbers, and possibly b different from zero.

The second expression (1:2) * 3 is a bit easier, because we could try to read the multiplcation as repetition (like we did in chapter 2): "3 times something is just something added to something added to something", which translates to the mathematical expression: (1:2) * 3 = (1:2) + (1:2) + (1:2) and I have a way to do additions between un-calculated divisions, two by two: (1:2) + (1:2) = (1+1):2 (1:2) + (1:2) + (1:2) = (1+1):2 + (1:2) = (1+1+1):2 and in a more general case, with n being any whole number: (1:2) * n = (1:2) + ... + (1:2) = (1+...+1):2 = (1*n):2
OK, so in general I have a rule like: (a:b) * n = (a*n):b which looks right, considering our examples so far.
Interestingly, this last rule reminds of the the inverse relation between multiplcation and division. Let's take for example 2, and a division like 1:2; if I multiply them I get 1, because: (1:2) * 2 = (1*2):2 = 1 and this is true for any whole number (excluding zero, which works very badly with divisions). So I can write a general property of multiplcation (and division) that says: (1:n) * n = 1 for any number n (just not zero). In our example here, n is 2 and 1:n is 0.5, so in mathematical jargon I could say that the number 0.5 is the inverse of the number 2, because 0.5 * 2 = 1!

So far so good. Now, the third expression is: (1:2) * (3:4). Well... I find this one very difficult to see. How can I read it so it even makes sense? And how could I rewrite it in the hope to solve it?
I might try with one of my nice calculation diagrams: on the left I do first the divisions and then the addition, and on the right I try to do the opposite...

Let's stop for a second and re-consider a simpler case, for example: 5 * (3:4). The most direct way to calculate this is to do the division first, then the multipilcation: 5 * (3:4) = 5 * 0.75 = 3.75 In fact, thanks to the properties of multiplication and division, the brackets can be taken off, and the expression should be exactly the same (AKA it should mean the same, but look different!): 5 * (3:4) = 5 * 3 : 4 = (5*3) : 4 = 15:4 = 3.75 which gives the same result. So, trusting that this is true in general, I could make up a rule like this: a * (b:c) = a * b : c = (a*b):c which again is totally reasonable, remembering the properties of multiplication, and what we did before for (1:2) * 3, that can also be written 3 * (1:2).

So now that I am convinced that a*(b:c) = (a*b):c, I can try to use it on my expression (1:2) * (3:4), where a will be the entire first bracket (1:2)! If I do that I should get: (1:2) * (3:4) = ( (1:2) *3 ):4
But ( (1:2) * 3 ):4 is made of two parts: (1:2) * 3 and (smtg):4. The first part is a known kind of expression, (a:b) * n = (a*n):b, or if your prefer it is the multiplication of a whole number by an un-calculated division, so I know what to do: (1:2) * 3 = (1*3):2 = 3:2 and the second part becomes: (smtg):4 = (3:2):4 ... and I'm stuck again. What is the meaning of (3:2):4? I'm dividing 3 by 2 and the result by 4, right?! But what does it the mean, what situation is this division of division describing?
OK, let's say I take a flock of sheeps and divide them between 2 friends. Then I take whatever each friend got and divide that among four relatives; how many sheeps will each one of the four relatives get?
Or if your prefer the "classic" cake example: take one cake, cut it in two. Then cut each half in four parts... how small is each of the resulting slices?

When you think about this kind of problems for a while your will realize that surely the order in which you divide is NOT IMPORTANT, what counts is the overall effect. So dividing by 2 and then by 4 has to be the same as dividing by 4 first and by 2 later! After all, if I divide a cake in two and each half in four, I will get 2*4 = 8 slices, and each will be one eighth of the whole cake.
And if that is true, then I can finish my calculation: (3:2):4 = (3:4):2 and they are both equal to 3:(2*4) = 3:8. And in fact all three expressions have the same result: 0.375.
So putting it all together: (1:2) * (3:4) = ( (1:2) *3 ):4 = ( (1*3):2 ):4 =
( 3:2 ):4 = 3:( 2*4 ) = 3:8 = 0.375 Hence, the diagram becomes:

and since the diagrams gives the same result no matter which order I follow to calculate, it should mean that the two ways are equivalent. I might be wrong, but let's try to write a general rule for these kinds of expressions: (a:b) * (c:d) = (a*c) : (b*d) which by the way looks kind of cool! Just move some of the variables and swap the operation signs... Cool.
But can I trust that it is true for any numbers a, b, c and d (eventually remembering that b and d cannot be zero)?
Well, one way could be to check that it works when b and d are equal to one, because in that case we are not really dealing with divisions anymore, so we should know what to expect. Let's see: (a:1) * (c:1) = (a*c) : (1*1) in other words: a*c = a*c which is surely correct! Good, so our rule at least does not make mistakes when numbers are involved, instead of actualy divisions. Nice.

Too long, didn't read...

OK, that was a bit too long... What about a more direct, more visual way to understand what (a:b) * (c:d) means and how to calculate it?
Sure, we can try with geometry. First we make up a way to draw a division, as a series of regular cuts in a rectangle, then we reason on the geometry of this representation of divisions. Finally, we can try to see what happens when we multiply two of these cuts together... For example, let's consider again: (1:2) * (3:4). In particular I could take 1:2 and draw it as half of a square (see below). Then I can draw 1:2 as three parts out of four, in a similar square.

Since the square is four by four, we are really looking at 16 * (1:2) and 16 * (3:4), and to the right (in the figure above) the square that repesents 16 * (1:2) * (3:4), which looks very much like 16:2 = 8 and then (8*3):4 = 24:4 = 6, and in fact the blue rectangle to the right in the figure above has six dots. I could try to first calculate 16 * (3:4), and then divide by two...

The figure above shows the four by four square where I first calculate 3:4, then on the remaining area I divide by two: 16 * (3:4) * (1:2), and I get (16*3):4 = 48:4 = 12 and then 12:2 = 6, and in fact the green rectangle to the right of the figure above has six dots.
And six is just what our rule would give as a result: (a:b) * (c:d) = (a*c) : (b*d) (1:2) * (3:4) = (1*3) : (2*4) = 3:8 and 16 * (3:8) = (16*3) : 8 = 48:8 = 6 ... bingo! I think.

Enter the fraction

I know now that I can be lazy when calculating divisions, and instead I can use various rules to work with un-calculated divisions in my expressions as if they where numbers; I might then just have to do a single division at the end of the calculation, or not even that, if you let me call "result" a division like 7:2.
Since an un-calculated division like 7:2 is just two numbers grouped together, I can decide to write it down as: 72 and remember that what I mean is "the first number divide by the second". This box notation can be useful later when I want to write down the rules for working with these un-calculated divisions.
Boxes are fine, but I would like to treat something like a:b as a single thing, as an actual number. So why not giving this things a special syntax... and a better name? How about calling an un-calculated divison a fraction?!
OK,fine. Now: to work as if a fraction was a number, I have to define how to add, multiply (and perhaps also subtract and divide) fractions with other fractions. Luckily we just did that in this chapter, only that we did it writing fractions as un-calculated divisons.
If I rewrite the rules we found with the syntax of fractions, I get:

  • 102 + 202 = 10 + 202 = 302
    Using the boxes the calculation becomes: 102+202 = 10+202 = 302
  • 12 + 3 = 1 + (3*2)2 = 1 + 62 = 72
    Using boxes: 12+3 = 1 + 3*22 = 72
  • 12 * 3 = 1 * 32 = 32
    And using boxes: 12*3 = 12*31 = 1*32*1 = 32
  • 12 * 34 = 1 * 32 * 4 = 38
    Witht the boxes: 12*34 = 1*32*4 = 38

Fine, but I also have to ask the usual question: is there a normal form for a fraction and what are the possible transformations that would change how it looks but not what it means?
Let's start with our friend un-calculated division 1:2 and write it as a fraction: 12. What is means is: "a division between two numbers, where the dividing number (at the bottom of the fraction) is two times as large as the number being divided (the one on the top of the fracton)". We know that if calculated, the division 1:2 is the number 0.5. And putting these two things together one could ask: are the other ways to write the same fraction? Answer: you bet!
For example 1020 is also 10:20 = 0.5 and that means it is ALSO a way to write 12. Let's see a few other examples: 12 , 24 , 36 , 1020 , 0.10.2 , 0.51 , -1-2 , -2-4 , ... , SZSSZ , ... OK, so ALL of these fractions ARE 12: so to be 12 you just need to be a fraction with some number on top and double as much at the bottom. And by "ten twentieth IS one half" here we mean: if you were to do the division, you will get the SAME result; we are talking about same meaning (semantics), even if the look (syntax) can be very different.
Sure. But can I choose one way to write 12 as "the official representative of the one half-ness"? That would be the normal form for that fraction. For me a normal form should be a simple, possibly beautiful way to write something, not too weird and that kind of conveys the meaning as clearly as possible. Here, 12 seems the simplest way to write this fraction: it has after all the smallest two numbers of all the others. Well, that is not entirely true: 0.10.2 has smaller numbers, but... they are uglier! Mainly because of the broken numbers. So perhaps the principle here could be: the normal form of a fraction should be a fraction with the same meaning (AKA the "same" fraction), but with the two smallest possible whole numbers. That gives me simple AND beautiful (or at least "not too ugly").
Note: surely I also need a fraction to never have the bottom number equal to zero, because a fraction is just a lazy division, and we know from the previous chapter that dividing by zero does not make sense.
OK, let's say that my definition of what a normal form is, makes sense. How do I check that a fraction is in normal form? And what can I do to bring a fraction to its normal form, in case it was not?
I can go back and try to find similarities among all the ways to write 12. In fact they all look like: k * 1k * 2 for some number k. Let me try with some small values of k, to see what happens.
When k=1 → 12, when k=3 → 36, and when k=0.1 → 0.10.2, and finally when k=-1 → -1-2.
Fine, this is an interesting property of fractions: "a fraction stays the same if I multiply the top and bottom numbers by the same value" (provided that value is NOT zero). And this property makes sense also looking at a fraction as a division, so no suprises there: in fact 1:2 = (1*5):(2*5) = 5:10 = 0.5.

Moral of the story so far: "12" is really the name of any fraction in the infinite family of fractions that are THE SAME as 1:2. I can write the same fraction ab in infinitely many other ways, but they will all look like k * ak * b, for some number k.

Special fractions and the zoom transformation

I think that some fractions are special, for example: 51 is a fraction but looks suspiciously like a normal whole number. And in fact if I look at it as a lazy division, it must be the same as 5:1 = 5, so a fraction with one at the bottom is a whole number.
Good, and what about 82? Surely that is four! But it is also fraction... not in normal form though. If I wanted to simplify this fraction, I could try to see it as something of the form: k * ak * b. Or if you like, I can try to divide both top and bottom numbers by the same (whole) number and see if I get simpler (but still whole) values for both: 82 → I try with 2 and get 41 and I could not go any further because four and one have no factors in common. A different way to see this process of simplification is to rewrite top and bottom number as prime factors: 82 → 2*2*22 → 2*21 and then remove common factors from the top and the bottom at the same time. Interestingly, that means that the simplification of a fraction stops when the two numbers (top and bottom) no longer have any common prime factors. In math they say that the two numbers are co-primes, or that they are prime with respect to each other.
So some fractions are really whole numbers, and other fractions can look like fractions but are really simpler fractions or even whole numbers as well. Proper, improper and apparent fractions Interestingly at this point we have also worked with a transformation of fractions that changes the look but leaves the meaning unchanged: ab → k * ak * b where k is any whole number, different from zero. I propose to call this transformation "zoom", so I can say "1020 is the fraction 12 zoomed by 10".
An alternative way to understood fractions could be to look at them as simply pairs of numbers with some special operations associated to them. In this sense, a fraction is a pair of whole numbers ab, where b is never zero, that together represent a lazy division; it is OK to transform both numbers together like this: ab → k*ak*b according to our zoom transformation, because the resulting lazy division (AKA fraction) is still the same. The normal form is when the two numbers a and b don't have any prime factors in common (AKA when they are co-primes). The operations that we normally do with numbers can be extended to work with fractions and even numbers and fractions. These operations can be written as rules that re-write fractions and numbers into a new fraction: ab + cb → a+cb ab * cd → a*bc*d a + bc → a1 + bc → a*c1*c + bc → a*c+bc a * bc → a1 * bc → a*b1*c → a*bc And finally, a fraction has an inverse of itself, for example: 23 is the inverse of 32, becuase when I multiply the first by the second I get 1.

Check that it is the case. Use the multiplication rule to calculate 23 * 32.
Whole numbers also have inverses, and those can be written as fractions; for instance: the inverse of 4 is 0.25, because 4 * 0.25 = 1, but 0.25 is also 14, and saying "the inverse of 4 is 14" is much more elegant and clear than using 0.25!
Note: zero is the only number that has no inverse.
